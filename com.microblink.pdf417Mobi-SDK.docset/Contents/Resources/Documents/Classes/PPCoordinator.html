PPCoordinator Class Reference

Inherits from NSObject
Declared in PPCoordinator.h

Overview

This object is the mastermind of the recognition process.

PPCoordinator coordinates hardware control with the recognition algorithms, and provides facotry method for creating ViewController with UI for the camera.

isScanningUnsupported:

This method returns true when scanning is unsupported on a specific device. Error object contains description of the reason for that.

+ (BOOL)isScanningUnsupported:(NSError **)error

Parameters

error

If scanning is not supported, when method this method returns, this parameter contains an NSError object that describes the problem. If you are not interested in possible errors, pass in NULL.

Return Value

YES if scanning is not supported, NO otherwise.

Discussion

This method returns true when scanning is unsupported on a specific device. Error object contains description of the reason for that.

View Controller which is presented on top of scanning view controller as a child.

Return Value

Scanning view controller fully initialized for presenting on screen.

Discussion

Method creates a camera view controller which is responsible for displaying the camera input on the phone screen. Also, camera view controller delivers PhotoPay results via PPPhotoPayDelegate object

With this method you can specify custom overlay view to be used on the camera display. The only requirement for the overlay view is that it’s a subclass of PPOverlayViewController
